# TA0002_Execution
공격자가 로컬 또는 원격 시스템을 통해 악성코드를 실행하기 위한 행동.

실행은 로컬 또는 원격 시스템에서 공격자가 제어하는 코드를 실행하는 기술로 구성됩니다. 악성 코드를 실행하는 기술은 종종 다른 모든 전술의 기술과 결합되어 네트워크 탐색이나 데이터 도용과 같은 더 광범위한 목표를 달성합니다. 예를 들어 공격자는 원격 액세스 도구를 사용하여 원격 시스템 검색을 수행하는 PowerShell 스크립트를 실행할 수 있습니다.

|T1059	|명령 및 스크립팅 인터프리터	|공격자는 명령, 스크립트 또는 바이너리를 실행하기 위해 명령 및 스크립트 인터프리터를 남용할 수 있습니다. 이러한 인터페이스와 언어는 컴퓨터 시스템과 상호 작용하는 방법을 제공하며 다양한 플랫폼에서 공통적인 기능입니다. 대부분의 시스템에는 명령줄 인터페이스와 스크립팅 기능이 내장되어 있습니다. 예를 들어 macOS 및 Linux 배포판에는 Unix Shell 의 일부가 포함되어 있는 반면 Windows 설치에는 Windows Command Shell 및 PowerShell 이 ​​포함 됩니다.|
|.001	|파워쉘	|공격자는 실행을 위해 PowerShell 명령과 스크립트를 남용할 수 있습니다. PowerShell은 Windows 운영 체제에 포함된 강력한 대화형 명령줄 인터페이스 및 스크립팅 환경입니다. 공격자는 PowerShell을 사용하여 정보 검색 및 코드 실행을 비롯한 다양한 작업을 수행할 수 있습니다. 예로는 Start-Process실행 파일을 실행하는 데 사용할 수 있는 cmdlet과 Invoke-Command로컬 또는 원격 컴퓨터에서 명령을 실행하는 cmdlet이 있습니다(하지만 PowerShell을 사용하여 원격 시스템에 연결하려면 관리자 권한이 필요함).|
|.002	|애플스크립트	|공격자는 실행을 위해 AppleScript를 남용할 수 있습니다. AppleScript는 AppleEvents라는 애플리케이션 간 메시지를 통해 애플리케이션과 OS의 일부를 제어하도록 설계된 macOS 스크립팅 언어입니다. 이러한 AppleEvent 메시지는 AppleScript를 사용하여 독립적으로 또는 쉽게 스크립트로 보낼 수 있습니다. 이러한 이벤트는 열려 있는 창을 찾고, 키 입력을 보내고, 열려 있는 거의 모든 응용 프로그램과 로컬 또는 원격으로 상호 작용할 수 있습니다.|
|.003	|Windows 명령 셸	|공격자는 실행을 위해 Windows 명령 셸을 남용할 수 있습니다. Windows 명령 셸( cmd )은 Windows 시스템의 기본 명령 프롬프트입니다. Windows 명령 프롬프트는 시스템의 거의 모든 측면을 제어하는 ​​데 사용할 수 있으며 명령의 여러 하위 집합에 필요한 다양한 권한 수준이 있습니다. 명령 프롬프트는 SSH 와 같은 원격 서비스 를 통해 원격으로 호출할 수 있습니다 .|
|.004	|유닉스 쉘	|공격자는 실행을 위해 Unix 셸 명령과 스크립트를 남용할 수 있습니다. Unix 셸은 Linux 및 macOS 시스템의 기본 명령 프롬프트이지만 특정 OS 또는 배포판에 따라 Unix 셸의 다양한 변형(예: sh, bash, zsh 등)이 있습니다. Unix 셸은 상승된 권한이 필요한 특정 명령으로 시스템의 모든 측면을 제어할 수 있습니다.|
|.005	|비주얼 베이직	|공격자는 실행을 위해 Visual Basic(VB)을 남용할 수 있습니다. VB는 Windows API를 통해 구성 요소 개체 모델 및 기본 API 와 같은 많은 Windows 기술과 상호 운용성으로 Microsoft에서 만든 프로그래밍 언어입니다 . 계획된 향후 발전이 없는 레거시로 태그가 지정되었지만 VB는 .NET Framework 및 플랫폼 간 .NET Core에서 통합 및 지원됩니다.|
|.006	|파이썬	|공격자는 실행을 위해 Python 명령과 스크립트를 남용할 수 있습니다. Python은 많은 기능을 수행할 수 있는 매우 인기 있는 스크립팅/프로그래밍 언어입니다. Python은 명령줄에서( python.exe인터프리터를 통해) 대화식으로 실행하거나 작성하고 다른 시스템에 배포할 수 있는 스크립트(.py)를 통해 실행할 수 있습니다. Python 코드는 바이너리 실행 파일로 컴파일할 수도 있습니다.|
|.007	|자바스크립트	|공격자는 실행을 위해 JavaScript의 다양한 구현을 남용할 수 있습니다. JavaScript(JS)는 일반적으로 웹 페이지의 스크립트와 관련된 플랫폼 독립적 스크립팅 언어(런타임에 JIT로 컴파일됨)이지만 JS는 브라우저 외부의 런타임 환경에서 실행할 수 있습니다.|
|.008	|네트워크 장치 CLI	|공격자는 악성 명령 및 페이로드를 실행하기 위해 네트워크 장치에서 스크립팅 또는 내장 명령줄 해석기(CLI)를 남용할 수 있습니다. CLI는 사용자와 관리자가 시스템 정보를 보거나 장치 작업을 수정하거나 진단 및 관리 기능을 수행하기 위해 장치와 상호 작용하는 기본 수단입니다. CLI에는 일반적으로 다양한 명령에 필요한 다양한 권한 수준이 포함되어 있습니다.|
|T1609	|컨테이너 관리 명령	|공격자는 컨테이너 관리 서비스를 남용하여 컨테이너 내에서 명령을 실행할 수 있습니다. Docker 데몬, Kubernetes API 서버 또는 kubelet과 같은 컨테이너 관리 서비스를 사용하면 환경 내에서 컨테이너를 원격으로 관리할 수 있습니다.|
|T1610	|컨테이너 배포	|공격자는 실행을 용이하게 하거나 방어를 회피하기 위해 환경에 컨테이너를 배포할 수 있습니다. 경우에 따라 공격자는 맬웨어를 실행하거나 다운로드하는 프로세스와 같이 특정 이미지 또는 배포와 관련된 프로세스를 실행하기 위해 새 컨테이너를 배포할 수 있습니다. 다른 경우에는 공격자가 환경 내의 기존 방어를 우회하기 위해 네트워크 규칙, 사용자 제한 등 없이 구성된 새 컨테이너를 배포할 수 있습니다.|
|T1203	|클라이언트 실행을 위한 악용	|공격자는 클라이언트 애플리케이션의 소프트웨어 취약점을 악용하여 코드를 실행할 수 있습니다. 예상치 못한 동작으로 이어질 수 있는 안전하지 않은 코딩 관행으로 인해 소프트웨어에 취약점이 존재할 수 있습니다. 공격자는 임의 코드 실행을 위한 표적 공격을 통해 특정 취약점을 이용할 수 있습니다. 종종 공격적인 툴킷에 대한 가장 가치 있는 익스플로잇은 원격 시스템에 대한 액세스 권한을 얻는 데 사용될 수 있기 때문에 원격 시스템에서 코드 실행을 획득하는 데 사용할 수 있는 익스플로잇입니다. 사용자는 일반적으로 작업을 수행하는 데 사용하는 응용 프로그램과 관련된 파일을 볼 것으로 예상하므로 높은 유용성으로 인해 익스플로잇 연구 및 개발에 유용한 대상입니다.|
|T1559	|프로세스 간 통신	|공격자는 로컬 코드 또는 명령 실행을 위해 IPC(프로세스 간 통신) 메커니즘을 남용할 수 있습니다. IPC는 일반적으로 프로세스에서 데이터를 공유하거나, 서로 통신하거나, 실행을 동기화하는 데 사용됩니다. IPC는 또한 프로세스가 순환 대기 패턴에 갇혀 있을 때 발생하는 교착 상태와 같은 상황을 피하기 위해 일반적으로 사용됩니다.|
|.001	|구성 요소 개체 모델	|공격자는 로컬 코드 실행을 위해 Windows 구성 요소 개체 모델(COM)을 사용할 수 있습니다. COM은 하나 이상의 인터페이스를 구현하는 실행 코드 또는 소프트웨어 개체 간의 상호 작용을 가능하게 하는 기본 Windows API(응용 프로그래밍 인터페이스)의 IPC(프로세스 간 통신) 구성 요소입니다. COM을 통해 클라이언트 개체는 일반적으로 바이너리 동적 링크 라이브러리(DLL) 또는 실행 파일(EXE)인 서버 개체의 메서드를 호출할 수 있습니다. 원격 COM 실행은 DCOM( 분산 구성 요소 개체 모델 ) 과 같은 원격 서비스 에 의해 촉진됩니다 .|
|.002	|동적 데이터 교환	|공격자는 Windows DDE(동적 데이터 교환)를 사용하여 임의의 명령을 실행할 수 있습니다. DDE는 응용 프로그램 간의 일회성 및/또는 지속적인 IPC(프로세스 간 통신)를 위한 클라이언트-서버 프로토콜입니다. 링크가 설정되면 애플리케이션은 문자열, 웜 데이터 링크(데이터 항목 변경 시 알림), 핫 데이터 링크(데이터 항목 변경 사항 복제) 및 명령 실행 요청으로 구성된 트랜잭션을 자율적으로 교환할 수 있습니다.|
|.003	|XPC 서비스	|공격자는 로컬 코드 실행을 위해 XPC 서비스 데몬에 악성 콘텐츠를 제공할 수 있습니다. macOS는 XPC 서비스 데몬과 타사 응용 프로그램 권한 있는 도우미 도구 간과 같은 다양한 프로세스 간의 기본적인 프로세스 간 통신에 XPC 서비스를 사용합니다. C API응용 프로그램은 낮은 수준의 XPC 서비스 또는 높은 수준 NSXPCConnection API의 권한이 필요한 작업(예: 네트워크 연결)을 처리하기 위해 루트로 실행되는 XPC 서비스 데몬에 메시지를 보낼 수 있습니다 . 응용 프로그램은 XPC 서비스의 청사진 역할을 하는 프로토콜 정의를 제공할 책임이 있습니다. 개발자는 일반적으로 XPC 서비스를 사용하여 애플리케이션 안정성과 애플리케이션 클라이언트와 데몬 간의 권한 분리를 제공합니다.|
|T1106	|네이티브 API	|공격자는 기본 OS API(응용 프로그래밍 인터페이스)와 상호 작용하여 동작을 실행할 수 있습니다. 네이티브 API는 하드웨어/장치, 메모리 및 프로세스와 관련된 것과 같은 커널 내에서 하위 수준 OS 서비스를 호출하는 제어된 수단을 제공합니다. 이러한 기본 API는 시스템 부팅 동안(다른 시스템 구성 요소가 아직 초기화되지 않은 경우) OS에 의해 활용될 뿐만 아니라 일상적인 작업 중에 작업 및 요청을 수행합니다.|
|T1053	|예약된 작업/작업	|공격자는 작업 예약 기능을 악용하여 악성 코드의 초기 또는 반복 실행을 용이하게 할 수 있습니다. 모든 주요 운영 체제에는 지정된 날짜와 시간에 실행될 프로그램이나 스크립트를 예약하는 유틸리티가 있습니다. 적절한 인증이 충족되는 경우 원격 시스템에서 작업을 예약할 수도 있습니다(예: Windows 환경에서 RPC 및 파일 및 프린터 공유). 원격 시스템에서 작업을 예약하려면 일반적으로 원격 시스템에서 관리자 또는 권한이 있는 그룹의 구성원이어야 합니다.|
|.002	|~에	|공격자는 at 유틸리티를 악용하여 악성 코드의 초기 또는 반복 실행을 위한 작업 스케줄링을 수행할 수 있습니다. at 유틸리티 는 지정된 시간과 날짜에 작업을 예약하기 위해 Windows, Linux 및 macOS 내에서 실행 파일로 존재합니다. Windows 환경에서 Scheduled Task 의 schtasks 를 위해 더 이상 사용되지 않지만 at 을 사용하려면 작업 스케줄러 서비스가 실행 중이고 사용자가 로컬 관리자 그룹의 구성원으로 로그온해야 합니다.|
|.003	|크론	|공격자는 이 유틸리티를 악용하여 cron악성 코드의 초기 또는 반복 실행을 위한 작업 스케줄링을 수행할 수 있습니다. 이 cron유틸리티는 Unix 계열 운영 체제를 위한 시간 기반 작업 스케줄러입니다. crontab파일에는 실행할 cron 항목의 일정과 지정된 실행 시간이 포함됩니다 . 모든 crontab파일은 운영 체제별 파일 경로에 저장됩니다.|
